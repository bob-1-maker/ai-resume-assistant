# AGENTS.md - AI 辅助开发协作规范

项目：AI 简历助手（Vue3 + Node.js 全栈）
模型：Codex
开发模式：测试先行 · 规范驱动 · 计划审核

1. 项目基本信息
   技术栈：
   前端：Vue3 + Vite + TypeScript + Element Plus + Vue Router + Axios
   后端：Node.js + Express + JWT 鉴权
   数据库：Turso
   AI 能力：硅基流动大模型 API（简历生成 / 解析 / 岗位匹配）
   核心模块：用户系统、权限拦截、简历管理、AI 调用、管理员后台
2. AI 开发工作流（强制顺序）
   确认需求 → 输出结构化需求文档
   生成 / 更新开发规范、目录结构、命名规则
   生成开发计划（Plan）→ 交由用户审核修改
   为目标功能编写：
   单元测试（Unit Test）
   端到端测试（E2E Test）
   根据测试用例开发功能实现
   运行测试，自动修正代码直到全部通过
   迭代优化规则与技能（Skill），补充到本文档
3. 代码规范
   文件名：小驼峰 / 短横线命名，语义化
   变量 / 函数：小驼峰，见名知意
   接口统一返回格式：{ code, msg, data }
   错误统一处理：400/401/403/500 状态码
   AI 接口必须：超时处理 + 重试机制 + 错误提示
   权限控制：JWT 携带、路由守卫、接口校验
4. 测试规范
   单元测试：覆盖核心逻辑、工具函数、接口逻辑
   E2E 测试：覆盖完整用户流程与异常流程
   必须包含：正常场景 + 空值 / 边界 + 权限 / 异常
5. 交付规范
   每次功能交付包含：测试代码 + 实现代码 + 使用说明
   不破坏原有功能，兼容现有登录、鉴权、路由结构
   代码可直接运行，无语法错误、无类型错误
6. Skill 记录（持续更新）
   登录鉴权与 JWT 处理
   路由守卫与权限拦截
   大模型 API 调用与稳定性优化
   表单验证与异常处理
   管理员后台权限隔离
   简历数据 CRUD 与导出
7. 初始开发计划（Development Plan）
   项目：AI 简历助手
   阶段一：基础稳定与权限完善
   修复现有 JWT 401 鉴权问题
   完善路由守卫，区分用户 / 管理员权限
   优化 AI 接口超时、重试、错误提示逻辑
   完成全流程手动测试，确保稳定
   阶段二：管理员后台模块（当前优先级最高）
   需求确认与需求文档
   编写单元测试：管理员登录校验、权限判断
   编写 E2E 测试：后台访问控制、简历列表、删除操作
   实现管理员登录页 /admin-login
   实现后台管理页 /admin
   实现独立管理员路由守卫
   测试通过后合并代码，确保不影响用户端
   阶段三：AI 调用稳定性优化
   增加请求重试机制
   统一超时时间与错误文案
   防止重复提交、防抖处理
   完善日志与用户提示
   阶段四：部署与上线前验证
   前后端统一环境配置
   全流程 E2E 测试
   部署上线与公网访问测试
   异常场景压测与修复
